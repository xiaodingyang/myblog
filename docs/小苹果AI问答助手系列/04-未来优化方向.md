# AI 问答助手未来优化方向：从向量检索到多轮对话

## 概述

本文探讨「小苹果」AI 问答助手的未来优化方向，包括向量检索、多轮对话、流式输出、用户反馈、全站知识图谱等技术进阶。通过这些优化，可以进一步提升问答的准确性和用户体验。

## 向量检索

### 为什么需要向量检索？

当前的关键词匹配检索存在以下局限：

1. **语义理解差**：同义词问题（"闭包" vs "closures"）无法匹配
2. **词形变化敏感**：时态、语态变化影响匹配（"write" vs "wrote"）
3. **上下文缺失**：无法理解句子和段落的完整语义

向量检索通过将文本转换为高维向量，计算向量间的余弦相似度，可以解决以上问题。

### 向量检索实现

```javascript
// 使用 OpenAI Embeddings API
const openai = require('openai');
const client = new openai.OpenAI({
  apiKey: process.env.AI_API_KEY,
});

async function generateEmbedding(text) {
  const response = await client.embeddings.create({
    model: 'text-embedding-3-small',
    input: text,
  });

  return response.data[0].embedding;
}

// 将段落向量化存储
async function vectorizeChunks(chunks) {
  const vectorized = [];
  
  for (const chunk of chunks) {
    const embedding = await generateEmbedding(chunk.text);
    vectorized.push({
      ...chunk,
      embedding,
      dimensions: 1536, // text-embedding-3-small 的维度
    });
  }
  
  return vectorized;
}
```

### 向量数据库选择

| 向量数据库 | 优势 | 劣势 |
|-----------|------|------|
| Pinecone | 托管、易用 | 成本高、锁定 |
| Qdrant | 开源、性能好 | 需要自部署 |
| Milvus | 功能丰富、性能强 | 部署复杂 |
| Weaviate | 向量+图、强语义 | 学习曲线陡 |

### 向量检索实现

```javascript
// 使用 Qdrant 作为向量数据库
const { QdrantClient } = require('@qdrant/js-client-rest');
const qdrant = new QdrantClient({ url: 'http://localhost:6333' });

async function vectorSearch(question, topK = 6) {
  // 1. 生成问题向量
  const questionEmbedding = await generateEmbedding(question);
  
  // 2. 向量搜索
  const searchResult = await qdrant.search('article_chunks', {
    vector: questionEmbedding,
    limit: topK,
    with_payload: true,
  });
  
  // 3. 返回结果
  return searchResult.points.map(point => ({
    text: point.payload.text,
    articleId: point.payload.articleId,
    score: point.score,
  }));
}
```

### 效果对比

| 检索方式 | 准确率 | 召回率 | 成本 |
|---------|--------|--------|------|
| 关键词匹配 | 60% | 70% | 低 |
| 向量检索 | 85% | 90% | 高 |
| 混合检索 | 80% | 85% | 中 |

**建议**：对于中小型项目，可以采用关键词匹配 + 向量检索的混合策略，平衡准确率和成本。

## 多轮对话

### 为什么需要多轮对话？

当前的问答是单轮的，每次提问都是独立的。多轮对话的优势在于：

1. **上下文记忆**：理解"它"、"刚才"、"后面"等指代
2. **连续追问**：基于之前的回答，深入探讨某个话题
3. **个性化体验**：记住用户的偏好和习惯

### 对话历史管理

```javascript
// 使用 Redis 存储对话历史
const redis = require('redis');
const client = redis.createClient();

async function getConversationHistory(userId, maxTurns = 5) {
  const key = `conversation:${userId}`;
  const history = await client.lrange(key, 0, maxTurns * 2);
  
  // 解析历史
  const turns = [];
  for (let i = 0; i < history.length; i += 2) {
    turns.push({
      role: history[i],
      content: history[i + 1],
    });
  }
  
  return turns;
}

async function addToConversationHistory(userId, role, content) {
  const key = `conversation:${userId}`;
  
  // 添加新消息
  await client.rpush(key, role);
  await client.rpush(key, content);
  
  // 限制历史长度
  await client.ltrim(key, -10); // 保留最近 5 轮
  
  // 设置过期时间（1 天）
  await client.expire(key, 86400);
}
```

```javascript
  // 1. 获取对话历史
  const history = await getConversationHistory(userId);
  
  // 2. 构建系统提示
  const systemPrompt = `你是本站助手「小苹果」。
  - 可以引用之前的对话内容
  - 如果用户问"它"、"刚才"等，指代之前提到的内容

  `;

    { role: 'system', content: systemPrompt },
    ...history,
    { role: 'user', content: message },
  ];
  
  // 4. 如果有文章上下文，添加到消息中

    const context = await retrieveContext(articleId, message);
    messages.push({
      role: 'system',
      content: `以下是与问题相关的文章内容：\n${context}`,
    });
  }
  
  // 5. 调用模型
  const answer = await callChatCompletions({ messages });
  
  // 6. 保存到对话历史
  await addToConversationHistory(userId, 'user', message);
  await addToConversationHistory(userId, 'assistant', answer);
  
  return {
    answer,
    history: [...history, { role: 'user', content: message }],
  };
}
```

## 流式输出

### 为什么需要流式输出？

当前的问答是等待完整回答后才返回给用户，这导致：

1. **等待时间长**：复杂问题可能需要 10-20 秒生成回答
2. **用户体验差**：用户不知道是否在处理
3. **无法中断**：无法中途停止生成

流式输出使用 SSE（Server-Sent Events）技术，实时推送生成的文本，打字机效果。

### SSE 流式输出实现

```javascript
// backend/src/controllers/aiChatController.js
exports.streamChat = async (req, res, next) => {
  try {
    const { message, articleId } = req.body;

    // 设置 SSE 响应头
    res.setHeader('Content-Type', 'text/event-stream');
    res.setHeader('Cache-Control', 'no-cache');
    res.setHeader('Connection', 'keep-alive');
    res.setHeader('X-Accel-Buffering', 'no');

    // 发送连接消息
    res.write(`: connected\n\n`);

    // 1. 获取对话历史
    const userId = getUserId(req);
    const history = await getConversationHistory(userId);

    // 2. 构建消息列表
    const messages = [
      { role: 'system', content: '你是本站助手「小苹果」...' },
      ...history,
      { role: 'user', content: message },
    ];

    // 3. 调用流式 API
    const stream = await openai.chat.completions.create({
      model: 'gpt-4o-mini',
      messages,
      stream: true,
    });

    // 4. 实时推送生成的文本
    let fullAnswer = '';

    for await (const chunk of stream) {
      const content = chunk.choices[0]?.delta?.content || '';
      if (content) {
        fullAnswer += content;
        
        // 推送文本块
        res.write(`data: ${JSON.stringify({ chunk: content })}\n\n`);
      }
    }

    // 5. 发送完成消息
    res.write(`event: done\ndata: ${JSON.stringify({ complete: true })}\n\n`);

    // 6. 保存到对话历史
    await addToConversationHistory(userId, 'user', message);
    await addToConversationHistory(userId, 'assistant', fullAnswer);

    res.end();

    console.log(`[Chat] 流式输出完成: ${fullAnswer.length} 字`);
  } catch (err) {
    res.write(`event: error\ndata: ${JSON.stringify({ error: err.message })}\n\n`);
    res.end();
    console.error('[Chat] 流式输出失败:', err);
    next(err);
  }
};
```

### 前端流式接收

```typescript
// frontend/src/hooks/useStreamChat.ts
import { useEffect, useState, useRef } from 'react';

interface StreamOptions {
  endpoint: string;
  message: string;
  articleId?: string;
}

export function useStreamChat({ endpoint, message, articleId }: StreamOptions) {
  const [content, setContent] = useState('');
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<Error | null>(null);
  const eventSourceRef = useRef<EventSource | null>(null);

  useEffect(() => {
    if (!message.trim()) return;

    setLoading(true);
    setError(null);
    setContent('');

    // 1. 创建 EventSource 连接
    const url = `${endpoint}?message=${encodeURIComponent(message)}${articleId ? `&articleId=${articleId}` : ''}`;
    const eventSource = new EventSource(url);
    eventSourceRef.current = eventSource;

    // 2. 监听消息
    eventSource.onmessage = (event) => {
      const data = JSON.parse(event.data);

      if (data.chunk) {
        // 追加文本块
        setContent(prev => prev + data.chunk);
      }

      if (data.complete) {
        // 完成标记
        setLoading(false);
        eventSource.close();
      }
    };

    // 3. 监听错误
    eventSource.onerror = (err) => {
      console.error('[Stream] EventSource error:', err);
      setError(new Error('连接失败，请稍后重试'));
      setLoading(false);
      eventSource.close();
    };

    // 4. 清理函数
    return () => {
      eventSource.close();
    };
  }, [endpoint, message, articleId]);

  return { content, loading, error };
}
```

## 用户反馈

### 为什么需要用户反馈？

用户反馈可以帮助我们：

1. **评估回答质量**：哪些回答有用，哪些不准确
2. **优化检索策略**：哪些检索结果相关，哪些不相关
3. **改进 Prompt**：哪些 Prompt 生成效果好，哪些需要调整
4. **收集新需求**：用户还希望有什么功能

```javascript
exports.getFeedbackStats = async (req, res, next) => {
  try {
    const stats = await Feedback.aggregate([
      {
        $group: {
          _id: '$helpful',
          count: { $sum: 1 },
        },
      },
      {
        $project: {
          helpful: '$_id',
          count: 1,
        },
      },
    ]);

    return res.json({
      code: 0,
      data: {
        total: stats.reduce((sum, s) => sum + s.count, 0),
        helpful: stats.find(s => s.helpful === true)?.count || 0,
        notHelpful: stats.find(s => s.helpful === false)?.count || 0,
        helpfulRate: stats.length > 0 
          ? stats.find(s => s.helpful === true)?.count / stats.reduce((sum, s) => sum + s.count, 0)
          : 0,
      },
    });
  } catch (err) {
    console.error('[Feedback] Stats failed:', err);
    next(err);
  }
};
```

## 全站知识图谱

### 为什么需要知识图谱？

知识图谱可以：

1. **建立文章关联**：基于标签、分类、引用关系构建图
2. **提升检索准确度**：通过图结构找到更相关的文档
3. **支持语义查询**：理解用户意图，找到隐含的相关内容
4. **可视化展示**：帮助用户理解知识结构

### 知识图谱构建

```javascript
// backend/src/services/knowledgeGraphService.js
class KnowledgeGraph {
  constructor() {
    this.nodes = new Map(); // 文章节点
    this.edges = new Map(); // 关系边
  }

  // 添加文章节点
  addArticleNode(article) {
    this.nodes.set(article._id, {
      id: article._id,
      title: article.title,
      type: 'article',
      category: article.category?.name,
      tags: article.tags || [],
      createdAt: article.createdAt,
    });
  }

  // 建立关系边
  addEdge(sourceId, targetId, type, weight = 1) {
    const edgeKey = `${sourceId}-${targetId}-${type}`;
    
    if (!this.edges.has(edgeKey)) {
      this.edges.set(edgeKey, {
        sourceId,
        targetId,
        type,
        weight,
        timestamp: new Date(),
      });
    }
  }

  // 构建图关系
  buildGraph(articles) {
    // 1. 添加所有文章节点
    for (const article of articles) {
      this.addArticleNode(article);
    }

    // 2. 基于分类建立关系
    const categoryGroups = new Map();
    for (const article of articles) {
      const categoryId = article.category?.id;
      if (categoryId) {
        if (!categoryGroups.has(categoryId)) {
          categoryGroups.set(categoryId, []);
        }
        categoryGroups.get(categoryId).push(article._id);
      }
    }

    for (const [categoryId, articleIds] of categoryGroups) {
      for (let i = 0; i < articleIds.length; i++) {
        for (let j = i + 1; j < articleIds.length; j++) {
          this.addEdge(articleIds[i], articleIds[j], 'category', 0.5);
        }
      }
    }

    // 3. 基于标签建立关系
    const tagGroups = new Map();
    for (const article of articles) {
      for (const tag of article.tags || []) {
        if (!tagGroups.has(tag)) {
          tagGroups.set(tag, []);
        }
        tagGroups.get(tag).push(article._id);
      }
    }

    for (const [tag, articleIds] of tagGroups) {
      for (let i = 0; i < articleIds.length; i++) {
        for (let j = i + 1; j < articleIds.length; j++) {
          this.addEdge(articleIds[i], articleIds[j], 'tag', 0.3);
        }
      }
    }

    // 4. 基于引用建立关系
    for (const article of articles) {
      const links = extractLinks(article.content); // 提取文章中的链接
      for (const link of links) {
        const linkedArticle = articles.find(a => a.url === link);
        if (linkedArticle) {
          this.addEdge(article._id, linkedArticle._id, 'link', 1);
        }
      }
    }
  }

  // 查找相关文章
  findRelatedArticles(articleId, { maxDepth = 2, maxResults = 5 } = {}) {
    const visited = new Set();
    const results = [];
    const queue = [{ id: articleId, depth: 0 }];

    while (queue.length > 0 && results.length < maxResults) {
      const { id, depth } = queue.shift();

      if (visited.has(id) || depth > maxDepth) {
        continue;
      }

      visited.add(id);

      // 找到所有相关的边
      for (const [key, edge] of this.edges) {
        if (edge.sourceId === id && !visited.has(edge.targetId)) {
          queue.push({ id: edge.targetId, depth: depth + 1 });
          
          if (depth > 0) {
            const node = this.nodes.get(edge.targetId);
            if (node && node.type === 'article') {
              results.push({
                ...node,
                relation: edge.type,
                weight: edge.weight,
              });
            }
          }
        }
      }
    }

    // 按权重排序
    results.sort((a, b) => b.weight - a.weight);

    return results.slice(0, maxResults);
  }
}

module.exports = KnowledgeGraph;
```

```javascript
const graph = new KnowledgeGraph();

// 构建知识图谱
async function buildKnowledgeGraph() {
  // 从数据库获取所有已发布的文章
  const articles = await Article.find({ status: 'published' })
    .select('_id title category tags content createdAt')
    .lean();

  // 构建图
  graph.buildGraph(articles);

  console.log(`[KnowledgeGraph] 构建完成: ${articles.length} 篇文章, ${graph.edges.size} 条关系`);
}

// 基于知识图谱的相关文章检索
async function findRelatedByGraph(articleId, maxResults = 5) {
  // 获取相关文章
  const related = graph.findRelatedArticles(articleId, { maxResults });
  
  // 提取相关段落
  const chunks = [];
  for (const article of related) {
    const articleDoc = await Article.findById(article.id);
    if (articleDoc) {
      const parts = splitContentIntoChunks(articleDoc.content);
      parts.forEach(p => {
        chunks.push({
          text: p,
          articleId: article.id,
          title: article.title,
          source: 'graph',
          relation: article.relation,
          weight: article.weight,
        });
      });
    }
  }

  return chunks;
}
```

### 知识图谱可视化

```typescript
// frontend/src/components/graph/KnowledgeGraph.tsx
import React, { useEffect, useRef } from 'react';
import { ForceGraph2D } from 'react-force-graph-2d';

interface Node {
  id: string;
  title: string;
  type: 'article' | 'category' | 'tag';
  val?: number;
}

interface Link {
  source: string;
  target: string;
  type: 'category' | 'tag' | 'link';
}

export const KnowledgeGraph: React.FC = () => {
  const [data, setData] = useState<{ nodes: Node[]; links: Link[] }>({
    nodes: [],
    links: [],
  });
  const graphRef = useRef();

  useEffect(() => {
    // 从后端获取知识图谱数据
    const fetchGraph = async () => {
      const res = await fetch('/api/ai/knowledge-graph');
      const graphData = await res.json();
      
      setData({
        nodes: graphData.data.nodes.map((node: any) => ({
          id: node._id,
          title: node.title,
          type: 'article',
          val: 10,
        })),
        links: graphData.data.edges.map((edge: any) => ({
          source: edge.sourceId,
          target: edge.targetId,
          type: edge.type,
        })),
      });
    };

    fetchGraph();
  }, []);

  return (
    <div className="knowledge-graph" style={{ width: '100%', height: '600px' }}>
      <ForceGraph2D
        ref={graphRef}
        graphData={data}
        nodeLabel="title"
        nodeAutoColorBy="type"
        linkDirectionalArrowLength={3.5}
        linkAutoColorBy="type"
        d3AlphaDecay={0.002}
        onNodeClick={(node) => {
          // 点击节点，跳转到文章页
          window.location.href = `/article/${node.id}`;
        }}
      />
    </div>
  );
};
```
