# 前端性能与工程化优化实战总结

## 项目背景

个人博客系统前端采用 **Umi 4 + React 18 + Ant Design 5 + Tailwind CSS** 技术栈，为提升用户体验和代码质量，进行了一系列性能优化和工程化改进。

## 一、打包体积优化

### 问题分析
- node_modules 体积达 1.4GB
- 首次加载资源过大，影响首屏性能
- 第三方库未做按需加载

### 优化方案

#### 1. 代码分割（Code Splitting）

在 `.umirc.ts` 中配置 Webpack 代码分割策略：

```typescript
export default defineConfig({
  // 代码分割优化
  chunks: ['vendors', 'umi'],
  chainWebpack(config) {
    config.optimization.splitChunks({
      chunks: 'all',
      cacheGroups: {
        antd: {
          name: 'antd',
          test: /[\\/]node_modules[\\/](antd|@ant-design)[\\/]/,
          priority: 20,
        },
        vendors: {
          name: 'vendors',
          test: /[\\/]node_modules[\\/]/,
          priority: 10,
        },
      },
    });
  },
});
```

**优化效果：**
- 将 Ant Design 单独打包，利用浏览器缓存
- 第三方库与业务代码分离，提升缓存命中率
- 减少单个 chunk 体积，加快首屏加载

#### 2. 懒加载（Lazy Loading）

对非首屏组件进行懒加载：

```typescript
// 粒子背景组件懒加载
const LazyParticlesBackground = lazy(() => import('@/components/ParticlesBackground'));
const LazyParticleThemeSelector = lazy(() => import('@/components/ParticleThemeSelector'));

// 使用 Suspense 包裹
<Suspense fallback={<Loading />}>
  {showParticles && <LazyParticlesBackground />}
</Suspense>
```

**优化效果：**
- 首屏不加载粒子动画，减少初始 JS 体积
- 按需加载，提升首屏渲染速度

## 四、工程化规范

### 1. EditorConfig 统一编码规范

创建 `.editorconfig` 文件，统一团队编码风格：

```ini
root = true

[*]
charset = utf-8
indent_style = space
indent_size = 2
end_of_line = lf
insert_final_newline = true
trim_trailing_whitespace = true

[*.md]
trim_trailing_whitespace = false
```

**收益：**
- 统一缩进、换行符等编码风格
- 减少代码审查中的格式争议
- 提升团队协作效率

### 2. 错误边界处理

实现 chunk 加载失败自动重试机制：

```javascript
headScripts: [
  `(function(){
    window.addEventListener('error', function(e) {
      var target = e.target || e.srcElement;
      if (target && target.tagName === 'SCRIPT' && /\\.async\\.js/.test(target.src || '')) {
        var key = 'chunk_retry_' + target.src;
        if (!sessionStorage.getItem(key)) {
          sessionStorage.setItem(key, '1');
          window.location.reload();
        }
      }
    }, true);
  })();`
]
```

**优化效果：**
- 解决 CDN 或网络问题导致的 chunk 加载失败
- 自动重试一次，避免白屏
- 使用 sessionStorage 防止无限刷新

## 五、常见问题排查与解决

### 1. Umi 配置兼容性问题

**问题描述：**
升级 Umi 4.6.34 后，使用 `chunks` 配置项导致启动失败：
```
Error: Invalid config keys: chunks
```

**原因分析：**
Umi 4 不支持 `chunks` 配置项，该配置在 Umi 3 中可用，但在 Umi 4 中已移除。

**解决方案：**
移除 `chunks` 配置，直接使用 `chainWebpack` 配置 Webpack：

```typescript
// ❌ 错误写法
export default defineConfig({
  chunks: ['vendors', 'umi'],
  chainWebpack(config) { ... }
});

// ✅ 正确写法
export default defineConfig({
  chainWebpack(config) {
    config.optimization.splitChunks({
      chunks: 'all',
      cacheGroups: {
        antd: {
          name: 'antd',
          test: /[\\/]node_modules[\\/](antd|@ant-design)[\\/]/,
          priority: 20,
        },
        vendors: {
          name: 'vendors',
          test: /[\\/]node_modules[\\/]/,
          priority: 10,
        },
      },
    });
  },
});
```

**面试要点：**
- 了解框架版本差异，避免使用废弃 API
- 遇到配置错误时，查阅官方文档确认配置项支持情况
- 使用 `chainWebpack` 可以直接操作 Webpack 配置，更灵活

## 八、面试要点

### 问题 1：如何优化 React 应用的首屏加载速度？

**回答要点：**
1. **代码分割**：使用 Webpack splitChunks 将第三方库单独打包
2. **懒加载**：React.lazy + Suspense 实现组件按需加载
3. **Tree Shaking**：移除未使用的代码
4. **CDN 加速**：静态资源使用 CDN 分发
5. **Gzip 压缩**：服务端开启 Gzip 压缩

### 问题 2：如何处理 chunk 加载失败的问题？

**回答要点：**
1. **监听 error 事件**：捕获 script 标签加载失败
2. **自动重试**：使用 sessionStorage 记录重试次数，避免无限刷新
3. **降级方案**：提示用户刷新页面或清除缓存

### 问题 3：如何做前端 SEO 优化？

**回答要点：**
1. **SSR/SSG**：服务端渲染或静态生成，提升首屏渲染速度
2. **Meta 标签**：完善 title、description、keywords
3. **语义化 HTML**：使用正确的 HTML 标签
4. **sitemap.xml**：生成站点地图，提交给搜索引擎
5. **robots.txt**：引导爬虫，禁止爬取敏感页面

### 问题 4：遇到依赖包升级后项目无法启动，如何排查？

**回答要点（结合实际案例）：**
1. **查看错误信息**：`ERR_REQUIRE_ESM` 表示 CommonJS 无法导入 ESM 模块
2. **分析依赖变化**：uuid 13.x 改为纯 ESM，不兼容 CommonJS 的 require
3. **解决方案**：
   - 降级到兼容版本（uuid 9.x）
   - 或改用 dynamic import
   - 或将项目改为 ESM（package.json 添加 `"type": "module"`）
4. **预防措施**：升级前查看 CHANGELOG，使用 `pnpm outdated` 检查大版本变更
