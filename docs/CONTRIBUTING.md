# 贡献指南

感谢你对本项目的关注！欢迎通过 Issue 和 Pull Request 参与贡献。

## 开发环境搭建

1. Fork 本仓库并克隆到本地
2. 按照 [README.md](./README.md) 中的「快速开始」步骤配置环境
3. 创建你的功能分支：`git checkout -b feature/your-feature`

## 提交规范

本项目使用以下 commit message 格式：

```
<type>: <description>
```

常用 type：

| type | 说明 |
|------|------|
| feat | 新功能 |
| fix | 修复 Bug |
| docs | 文档变更 |
| style | 代码格式（不影响功能） |
| refactor | 重构（不增加功能、不修复 Bug） |
| perf | 性能优化 |
| chore | 构建/工具/辅助变更 |

示例：`feat: 添加文章搜索功能`

## Pull Request 流程

1. 确保你的代码可以正常运行
2. 更新相关文档（如有必要）
3. 提交 PR 时请清晰描述你的改动内容和目的
4. 等待代码审查

## 代码风格

- 前端：遵循 ESLint 配置，使用 TypeScript
- 后端：遵循 ESLint 配置，使用 CommonJS 模块

## 报告 Bug

提交 Issue 时请包含：

1. Bug 的简要描述
2. 复现步骤
3. 预期行为与实际行为
4. 运行环境（浏览器、Node.js 版本等）

## 功能建议

欢迎通过 Issue 提出功能建议，请描述你期望的功能及其使用场景。
